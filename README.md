# 蓝牙音箱保活 🎵

Windows 后台应用，每隔指定秒数播放静音音频，保持蓝牙音响连接不断开。

## 问题

蓝牙音响在暂停播放后，系统会在几十秒后自动断开连接。每次需要重新手动连接，非常麻烦。

## 解决方案

程序在系统托盘后台运行，每隔 30 秒（可自定义）播放一次极短静音音频，让蓝牙音响保持"活跃"状态，系统就不会自动断开连接。

**全程无声**，不会打扰你。

## 功能特点

- ✅ **系统托盘运行** — 最小化到托盘，不占任务栏
- ✅ **自动保持连接** — 每30秒播放静音音频
- ✅ **开机自启** — 右键菜单一键设置
- ✅ **自定义间隔** — 可自由调节播放间隔
- ✅ **Windows原生对话框** — 稳定可靠，无事件循环冲突
- ✅ **独立exe** — 无需Python环境，双击即用

## 下载与使用

### 方式1：直接下载exe

从 [Releases 页面](https://github.com/Gvshow/-/releases) 下载 `蓝牙音响保持.exe`，双击运行即可。

### 方式2：从源码运行

```bash
# 1. 安装依赖
pip install pystray pillow

# 2. 运行
python bluetooth_keepalive_simple.py
```

## 使用方法

1. 双击运行程序
2. 程序自动最小化到**系统托盘**（右下角通知区域）
3. **右键**托盘中的喇叭图标：
   - **服务状态** — 查看运行状态、播放次数
   - **设置间隔** — 修改播放间隔（默认30秒）
   - **开机自启** — 启用/禁用开机自动运行
   - **查看日志** — 用记事本查看运行日志
   - **退出** — 关闭程序

## 配置说明

配置文件 `config.json`（与exe同目录）：

```json
{
  "interval": 30
}
```

也可以在程序内通过「设置间隔」菜单修改。

## 工作原理

| 步骤 | 说明 |
|:---|:---|
| ① | 启动定时器，每N秒触发一次 |
| ② | 播放极短静音音频（约0.3秒） |
| ③ | 蓝牙音响被激活，保持连接状态 |
| ④ | 系统判定设备"活跃中"，不会断开 |

### 三重音频保障

1. **`winsound` 系统声音** — Windows 内置，最可靠
2. **临时 WAV 静音文件** — 如果方法1失败
3. **PowerShell 蜂鸣声** — 最后保障

## 技术栈

- Python 3
- pystray（系统托盘）
- winsound（音频播放）
- ctypes（Windows原生对话框）
- PyInstaller（打包exe）
